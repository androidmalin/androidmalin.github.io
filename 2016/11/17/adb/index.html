Linux,Unix下环境变量的配置

发表于 2016-11-17 |

这里配置到当前的用户下

1.编辑配置文件

命令如下

vim ~/.bashrc

2.进入后，在英文输入法下，输入i(表示插入),在配置文件的最后，添加如下配置，这里以Android SDK Tool为例

export ANDROID_HOME=/home/malin/sdk

export PATH=$PATH:$ANDROID_HOME/platform-tools

export PATH=$PATH:$ANDROID_HOME/tools

说明:

这里/home/malin/sdk 是我电脑上Android SDK的路径。请替换为你自己电脑上Android SDK的路径。

便捷的方法是，使用cd命令切换到sdk的文件夹下。

例如：cd /home/malin/sdk。然后使用pwd命令查看当前的路径

插入如上的文本后，就是保存了,步骤如下所示:

a.按 Esc键（键盘的左上角）

b.英文输入法下输入 :wq

c.输入回车

这样紫就保存了.

3.使配置文件生效

使用如下的命令

source ~/.bashrc

4.测试是否生效

adb version

5.获取手机IP地址：

将 Android 设备与要运行 adb 的电脑连接到同一个局域网，比如连到同一个 WiFi。

将设备与电脑通过 USB 线连接。

应确保连接成功（可运行 adb devices 看是否能列出该设备）。

让设备在 5555 端口监听 TCP/IP 连接：

adb tcpip 5555

断开 USB 连接。

找到设备的 IP 地址。

一般能在「设置」-「关于手机」-「状态信息」-「IP地址」找到。

通过 IP 地址连接设备。

或者使用如下的脚本获取手机的IP地址。

adb shell ifconfig | grep "inet addr" | grep "Bcast" | cut -d ":" -f 2 | cut -d " " -f 1

adb connect

这里的 就是上一步中找到的设备 IP 地址。

确认连接状态。

adb devices

6.将/data/data/packagename/database/a.db拉到电脑上，应用为debugable=true

  1. adb exec-out run-as tv.panda.live.broadcast cat databases/panda_net_server.db > ./panda_net_server.db

  2. adb exec-out run-as tv.panda.live.broadcast cat files/cover/cover.jpg > ./cover.jpg

  3. adb exec-out run-as tv.panda.live.broadcast tar c databases/ > databases.tar,tar -xvf databases.tar

  4. adb exec-out

adb shell

run-as tv.panda.live.broadcast

cat files/cover/cover.jpg > /sdcard/cover.jpg

exit

exit

adb pull /sdcard/cover.jpg /home/malin/nexus6p/myconfig/backup

pwd

w3m path
